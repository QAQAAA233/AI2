## 🔐 八、關鍵變量傳遞路徑

| 變量 | 起點 | 傳遞路徑 | 終點 |
|------|------|----------|------|
| `folder_path` | 前端選擇 | Flask → ProcessManager → CodeProcessor | 文件系統 |
| `prompt` | 用戶輸入 | Flask → ProcessManager → GeminiAI | API請求 |
| `files[]` | 文件上傳 | Base64編碼 → GeminiAI | multimodal輸入 |
| `usage_metadata` | Gemini響應 | response.usage_metadata → ConversationMessage | 對話歷史 |
| `terminal_output` | subprocess.stdout | queue → ProgramManager → ConversationMessage | 前端顯示 |
| `is_iteration` | 前端狀態 | Flask → ProcessManager | 決定執行模式 |
| `project_dir` | 路徑計算 | 新建/迭代分支 | 最終項目位置 |

### 9.2 Token統計機制 (v5.5核心)

```python
# 從Gemini響應提取
if hasattr(response, 'usage_metadata'):
    um = response.usage_metadata
    usage_metadata = {
        'prompt_token_count': um.prompt_token_count,
        'candidates_token_count': um.candidates_token_count,
        'thoughts_token_count': um.thoughts_token_count,
        'total_token_count': um.total_token_count
    }

# 獨立保存到ConversationMessage
ConversationManager.add_message(
    project_dir,
    'assistant',
    content,
    usage_metadata=usage_metadata  # ★ 獨立字段
)
```

### 9.3 項目目錄管理邏輯

```python
# v5.5修復：確保路徑一致性
if is_iteration:
    final_project_dir = folder_path  # 迭代：使用現有路徑
else:
    final_project_dir = folder_path / project_name  # 新建：創建子目錄
    
    # 遷移對話記錄到新目錄
    temp_conversation = ConversationManager.load_conversation(folder_path)
    temp_conversation.project_dir = final_project_dir
    ConversationManager.save_conversation(temp_conversation)
    
    # 清理臨時文件
    temp_conv_file.unlink()
```

## ⚡ 十、性能優化要點

1. **異步輸出捕獲**：使用threading + queue避免阻塞
2. **智能視窗匹配**：多層匹配邏輯減少誤判
3. **Base64流式處理**：避免大文件內存溢出
4. **對話歷史壓縮**：只保留最近50行terminal輸出
5. **瀏覽器進程追蹤**：獨立管理避免殘留進程
